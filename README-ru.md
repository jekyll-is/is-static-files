# is-static-files

Кастомные наследники класса StaticFile для Jekyll.

## Обзор

is-static-files - это Ruby гем, который расширяет возможности обработки статических файлов в Jekyll за счёт создания кастомного класса StaticFile. Он позволяет работать со статическими файлами, которые могут иметь исходный файл или содержаться непосредственно в памяти, что удобно для генерации или модификации контента программным путём во время сборки сайта.

## Возможности

- Наследование от `Jekyll::StaticFile` для полной интеграции с Jekyll.
- Поддержка файлов с исходным файлом на диске.
- Поддержка файлов с содержанием из памяти, без исходного файла.
- Кастомные пути назначения и относительные пути.
- Совместимость с Jekyll версии 4.3 и выше и Ruby ~> 3.4.
- Наличие тестов на RSpec и отчёта покрытия SimpleCov.

## Установка

Добавьте в ваш Gemfile строки:

```ruby
gem 'is-static-files', '~> 0.8.0'
```

Затем выполните:

```shell
bundle install
```

Или установите гем глобально:

```shell
gem install is-static-files
```

## Использование

Создавайте объекты `IS::StaticFile`, чтобы включить в сборку Jekyll статические файлы.

```ruby
# Файл с исходным файлом на диске
file = IS::StaticFile.new(site, '/target/path/', 'example.txt', source: '/full/path/to/src/example.txt')

# Файл с контентом из памяти без исходного файла
file2 = IS::StaticFile.new(site, '/target/path/', 'generated.txt', content: 'Динамически сгенерированное содержимое.')

# Добавление файлов в генерацию сайта (например, в генераторе Jekyll)
site.static_files << file
site.static_files << file2
```

Метод `write` позаботится о корректной записи файлов в папку назначения при генерации сайта.

## Разработка и тестирование

Клонируйте репозиторий и выполните:

```shell
bundle install
rake spec
```

Это запустит тесты через RSpec с отчётом покрытия через SimpleCov.
